## 3. LEGAL & REGULATORY ALIGNMENT

### 3.1 Medical & Patient Data Responsibility

The system must explicitly define:

- **Data Controller:** The clinic / medical entity
- **Data Processor:** System operator or vendor (if external)

Rules:

- All patient and medical data are owned by the clinic
- Data may only be processed for clinical and operational purposes
- No reuse, resale, analytics resale, or secondary processing is allowed
- System operator access is limited to:
  - maintenance
  - technical support
  - legally required audits

---

### 3.2 Patient Rights (Operationally Enforced)

The system must support:

- Right to access patient data
- Right to correction (with full audit trail)
- Right to data export (medical summary)
- Right to access restriction via RBAC

**Hard Rule:**  
No medical or financial record may ever be physically deleted.  
All corrections must be logical and fully audited.

---

## 4. FINANCIAL COMPLIANCE & RECORD RETENTION

### 4.1 Financial Record Integrity

The following records are **legally sensitive and immutable**:

- Invoices
- Payments
- Installment payments
- Inventory purchase costs
- Expenses
- Profit and earnings reports
- Audit logs

Rules:

- No edits after creation
- Corrections only via VOID or compensating entries
- Every correction must:
  - have a reason
  - be attributable to a user
  - be audit-logged

---

### 4.2 Retention Policy (Baseline)

Unless local law requires otherwise:

| Record Type       | Minimum Retention Period |
| ----------------- | ------------------------ |
| Medical records   | As per medical law       |
| Financial records | 5–10 years               |
| Audit logs        | ≥ 5 years                |
| System logs       | ≥ 1 year                 |
| Backups           | ≥ 6 months               |

Retention configuration must be approved by clinic management.

---

## 5. DATA PROTECTION & ACCESS GOVERNANCE

### 5.1 RBAC Finalization

- RBAC rules defined in Phase 5 are **final**
- Any role change requires:
  - justification
  - audit logging
  - reversibility
- Emergency access (if allowed):
  - must be time-limited
  - must be audited
  - must be reviewed post-use

---

### 5.2 Data Exposure Limits

The system guarantees:

- Dentists cannot view clinic-wide financial data
- Only authorized roles can access financial records
- External integrations cannot access PII unless explicitly approved
- Reports respect visibility constraints strictly

## 7. CHANGE GOVERNANCE MODEL

### 7.1 Change Classification

Every future change must be classified as:

- Bug fix (no behavior change)
- Compliance-driven change
- Feature enhancement
- Breaking change

---

### 7.2 Approval Matrix

| Change Type | Required Approval |
| ----------- | ----------------- |
| Bug fix     | Technical Owner   |
| Compliance  | Legal + Manager   |
| Feature     | Manager + Owner   |
| Breaking    | Full Governance   |

No change proceeds without classification and approval.

---

## 8. OWNERSHIP & LIABILITY MATRIX

### 8.1 Ownership

| Area                  | Responsible Owner |
| --------------------- | ----------------- |
| Patient data          | Clinic            |
| Financial correctness | Clinic management |
| System configuration  | Manager           |
| Codebase              | Technical owner   |
| Backups               | Assigned owner    |
| Integrations          | Clinic + Vendor   |

---

### 8.2 Liability Boundaries

- Clinic is responsible for:
  - correctness of entered data
  - operational usage
  - staff access control
- System provider is responsible for:
  - system behavior as specified
  - data integrity mechanisms
  - auditability and safeguards

These boundaries must be documented and accepted.

---

## 9. AUDIT READINESS

### 9.1 Internal Audit Capability

The system must demonstrate:

- who accessed what and when
- all financial record creation and voiding
- inventory movements and costs
- report generation history
- backup existence and restore capability

---

### 9.2 External Audit Support

Provide:

- read-only audit access
- exportable audit logs
- documented financial and reporting definitions
